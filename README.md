# News RAG System with Ollama and LangChain
_Проект носит демонстрационный характер. Преследовалась цель создания устойчиво работающего MVP.
Структура папок проекта, маниакальная обработка ошибок и исключений, тесты и прочие best-practices временно отложены._

Проект представляет собой систему, которая автоматически собирает новости, превращает их в контекст LLM и использует для ответов на вопросы пользователя, как локальную базу знаний.

## Описание работы

Система состоит из двух параллельно работающих компонентов, организованных через треды:

**Фоновый сборщик новостей** автоматически парсит RSS новостного сайта с заданной периодичностью. Полученный XML RSS фида разбирается на ссылки, после чего парсится полный текст новости при помощи BeautifulSoup4. 

Перед парсингом полного текста новости используется дедупликация — если статья уже была добавлена ранее, она не будет обработана повторно. 

Текст каждой статьи разбивается на чанки через LangChain Text Splitters. Затем чанки преобразуются в векторные представления с помощью модели эмбеддингов и сохраняются в векторную базу данных Chroma.

**Интерфейс вопрос-ответ** работает в основном потоке и ожидает ввода пользователя в консоли. 

После получения запроса от пользователя, система ищет в векторной базе наиболее релевантные чанки, формирует контекст и отправляет его вместе с запросом в локальную LLM. 

Для генерации ответа используется RAG с продуманным промптом, реализующим подход few-shot + Chain-of-Thought (CoT). 

Промпт содержит четкую систему инструкций, два примера эталонных ответов (few-shot), демонстрирующих требуемый стиль аналитики, и жесткое требование дать прямой ответ, а затем развернутое объяснение контекста и причин, которые имплементирует CoT. 

Модель генерирует ответ исключительно на основе найденной информации, строго следуя заданному формату, без обращения к внешним API.

## Стек

- **Python 3.14** — основная среда разработки
- **LangChain** — фреймворк для работы с текстовыми сплиттерами и построения RAG
- **Ollama** — локальный сервер для запуска языковых моделей
- **BeautifulSoup4** — парсинг HTML-страниц и извлечение текста новостей
- **ChromaDB** — векторная база данных для хранения и поиска семантических соответствий
- **Requests** — загрузка данных с веб-сайтов и RSS-лент

## Возможности

- Автоматический парсинг в отдельном потоке 
- Дедупликация статей
- Разбиение на чанки с перекрытием
- Локальное векторное хранилище
- Консольный интерфейс для вопросов с потоковой генерацией ответов
- Полная приватность — все данные и модель работают локально
