# News RAG System with Ollama and LangChain
_Проект носит демонстрационный характер. Преследовалась цель создания устойчиво работающего MVP.
Структура папок проекта и прочие best-practices временно отложены._

Проект представляет собой интеллектуальную систему для работы с новостями, которая автоматически собирает новостные статьи с переданного url, превращает их в контекст LLM и использует для ответов на вопросы пользователя, как локальную базу знаний.

## Описание работы

Система состоит из двух параллельно работающих компонентов, организованных через треды:

**Фоновый сборщик новостей** автоматически парсит новостные сайты с заданной периодичностью. При парсинге используется дедупликация — если статья уже была добавлена ранее, она не будет обработана повторно. Текст каждой статьи очищается от HTML-тегов с помощью BeautifulSoup4 и разбивается на смысловые чанки с использованием LangChain Text Splitters. Затем чанки преобразуются в векторные представления с помощью модели эмбеддингов и сохраняются в векторную базу данных.

**Интерфейс для вопросов и ответов** работает в основном потоке и ожидает ввода пользователя в консоли. Когда вы задаете вопрос, система ищет в векторной базе наиболее релевантные фрагменты новостей, формирует контекст и отправляет его вместе с вопросом в локальную языковую модель. Модель генерирует ответ исключительно на основе найденной информации, без обращения к внешним API.

## Технологический стек

- **Python 3.14** — основная среда разработки
- **LangChain** — фреймворк для работы с текстовыми сплиттерами и построения RAG
- **Ollama** — локальный сервер для запуска языковых моделей
- **Qwen 2.5** — языковые модели для генерации ответов и создания эмбеддингов
- **BeautifulSoup4** — парсинг HTML-страниц и извлечение текста новостей
- **ChromaDB** — векторная база данных для хранения и поиска семантических соответствий
- **Requests** — загрузка данных с веб-сайтов и RSS-лент

## Возможности

- Автоматический сбор новостей с полным текстом новости через RSS-ленту
- Дедупликация статей
- Разбиение на чанки с перекрытием
- Локальное векторное хранилище
- Консольный интерфейс для вопросов с потоковой генерацией ответов
- Полная приватность — все данные и модель работают локально
